How To Create a Website on Cloudflare
25 March 2026
Note: this was written on 25 March 2026. Cloudflare's site changes quickly, but the gist of the steps should be the same.

Summary

  1. create Domain Name (link)
  2. upload or link to content by one of several methods:
    • using drag and drop
    • connecting to github
    • using Cloudflare Workers
    • pointing to AWS
  3. set up some DNS information (apex, redirects, etc)

Direct Upload (Drag And Drop)

Direct Upload enables you to upload your content to CloudFlare Pages and deploy them to the Cloudflare global network. You should choose Direct Upload over git integration if you want to integrate your own build platform or upload from your local computer.
Warning: if you choose Direct Upload, you cannot switch to Git integration later. You will have to create a new project with Git integration to use automatic deployments. I am not sure what this entails.
Note: alternatively, You can use cloudflare wrangler to do direct-upload from your computer.

  1. On cloudflare's dashboard page, on the left side bar, click on: Compute → Workers & Pages
  2. Click on the blue Create application button
  3. Click on the link Looking to Deploy Pages? Get started
  4. Click on Drag and drop your files Get started button
  5. enter yoursitename (e.g. www-yoursite-com) for the project name (it will be served on .pages.dev)
  6. drag and drop your content

To link <project_name>.pages.dev to your custom domain name add following DNS records:
Type Name Content Proxy status TTL
A WWW 192.0.2.1 proxied Auto
CNAME yoursite.com project_name.pages.dev proxied Auto

Using github

Warning: if you deploy using github you cannot switch to Direct Upload later.
Note: you can generate preview URLS for a commit to another branch in the repository without affecting your production deployment.
Note: if you do not want to trigger deployments every time you push a commit, you can disable automatic deployments or add [SI skip] to the commit message (see below).

  1. On cloudflare's dashboard page, on the left side bar, click on: Compute → Workers & Pages
  2. select create application -> pages -> connect to Git
    (this allows cloudflare to access your git repository)
  3. select your github repository
  4. select install & authorize and Begin Setup

To disable automatic builds on branches

  1. Compute&AI -> workers & pages -> go to your pages project
  2. go to Build -> edit branch cntrl -> turn off automatic branch deployments
  3. or change Preview Branch ot None (Disable automatic branch deployments)
You can also skip an individual cloudflare build by adding a prefix to the commig message: commit -m [CI skip] ...

Using Cloudflare Workers

Note: cloudflare workers also support Git integrations to automatically build and deploy Workers from your connected Git repository. Learn more in Workers Builds
  1. in git check-in a wrangler configuration file: wrangler.jsonc 
            {
          "name": "www-quirky-place",
          "compatibility_date": "2026-01-31",
          "assets": {
           "directory": "./public"
          }
        }
  2. go to cloudflare and create/use existing custom domain (www.quirky.place)
  3. dashboard -> Compute&AI -> Create application -> Pages -> Connect to Git
  4. pick your cloudflare user, select repo want access
  5. enter github user and use mobile phone auth code (or login) to verify
  6. then pick github againa dn select a repository
  7. add a custom domain to the worker/page under:
    compute & AI ->Workers & pages -> www-quirky-place settings tab
    domaind & routes +Add
    www.quirky.place
